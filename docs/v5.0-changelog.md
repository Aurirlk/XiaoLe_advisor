# V5.0 Changelog — 投研级志愿系统

> 发布日期: 2026-06-15

---

## 🎯 版本亮点

V5.0 是小乐AI的重大升级，从"会说话的表格"升级为"真正的投研系统"：

1. **Neo4j知识图谱** — 复杂关系精准查询，告别SQL多表联查
2. **家庭冲突检测** — 双向约束矩阵，冲突显性化
3. **反方审计引擎** — 5个视角挑刺，阻断AI讨好型幻觉
4. **AI暴露度评估** — 评估专业入门岗位被AI替代的风险
5. **调查问卷系统** — 快速完成画像，减少对话轮次
6. **量化评分框架** — 100分制7维度评分，决策透明化

## 📦 新增功能

### 1. Neo4j知识图谱集成

**新增文件：**
- `configs/neo4j_config.yaml` — Neo4j配置
- `scripts/import_neo4j.py` — 数据导入脚本
- `tools/neo4j_tools.py` — 3个图谱查询工具

**工具列表：**
| 工具 | 说明 |
|------|------|
| `query_neo4j_admission_tool` | 分数段院校推荐查询 |
| `query_career_path_tool` | 专业→职业路径查询 |
| `query_university_info_tool` | 大学详细信息查询 |

**图模型：**
```
(University)-[:LOCATED_IN]->(Province)
(University)-[:OFFERS {min_score, year}]->(Major)
(Major)-[:LEADS_TO]->(Career)
```

### 2. 家庭冲突检测器

**新增文件：** `skills/conflict_detector.py`

**检测维度：**
- 预算 vs 公办要求
- 城市偏好 vs 分数现实
- 专业偏好 vs 家长禁止
- 体检限制 vs 目标专业
- 读研意愿 vs 专业要求

**输出：**
- `has_conflict`: 是否存在冲突
- `requires_arbitration`: 是否需要家庭仲裁
- `resolution_prompt`: 冲突解决提示词

### 3. 反方审计引擎

**新增文件：** `skills/red_team_auditor.py`

**审计视角：**
1. 招生办视角：体检退档、分数不足
2. HR/就业视角：预算击穿、专业天坑
3. 家长视角：隐性成本、培养周期
4. 学生视角：兴趣错配、能力不足
5. 应届生视角：真实就业、发展路径

**红线规则库：**
- 高成本专业
- 体检严格专业
- AI高风险任务
- 天坑专业

### 4. AI暴露度评估器

**新增文件：** `skills/ai_exposure_checker.py`

**评估维度：**
- 高风险任务（易被替代）
- 高壁垒任务（不易被替代）
- 能力增强建议

**预设专业数据：**
- 计算机、软件、人工智能
- 会计、金融、法学
- 临床医学、土木工程
- 汉语言文学、英语等

### 5. 投研级数据表

**新增文件：** `data/sql_schema/10_major_stats.sql`

**四张去水表：**
1. `major_destination_stats` — 就业去向表
2. `major_salary_stats` — 收入证据表
3. `major_stability_stats` — 稳定性线索表
4. `major_career_path` — 发展路径表

### 6. 调查问卷系统

**新增文件：**
- `configs/questionnaire_config.yaml` — 问卷配置
- `skills/questionnaire_service.py` — 问卷服务层
- `api/routers/questionnaire_router.py` — 问卷API
- `frontend/components/QuestionnairePage.js` — 前端页面

**问卷类型：**
| 类型 | 说明 | 时长 |
|------|------|------|
| `student_basic` | 学生基础信息 | 3-5分钟 |
| `mbti_personality` | MBTI性格测试 | 8-10分钟 |
| `parent_constraints` | 家长意见采集 | 3-5分钟 |
| `major_preference` | 专业意向调查 | 5-8分钟 |

**支持题型：**
- 单选题
- 多选题
- 填空题
- 论述题

**API端点：**
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/questionnaire/types` | 获取问卷类型列表 |
| GET | `/questionnaire/{type}` | 获取问卷详情 |
| POST | `/questionnaire/validate` | 验证问卷答案 |
| POST | `/questionnaire/submit` | 提交问卷 |

### 7. 量化评分框架

**新增文件：** `skills/quantitative_scorer.py`

**7个评分维度（满分100分）：**
| 维度 | 分值 | 说明 |
|------|------|------|
| 录取风险 | 25分 | 位次匹配度、录取把握 |
| 专业适配 | 20分 | 兴趣匹配、学科基础 |
| 就业钱景 | 18分 | 市场容量、薪资区间 |
| 城市产业 | 12分 | 产业集群、实习便利 |
| 学校平台 | 10分 | 学科声誉、保研比例 |
| AI暴露 | 10分 | 技术替代、增强潜力 |
| 家庭共识 | 5分 | 冲突解决、意见一致 |

## 🔧 架构变更

### GraphState 扩展

```python
# 新增字段
parent_constraints: ParentConstraints   # 家长硬约束
student_preferences: StudentPreferences # 学生软偏好
family_conflict: Dict[str, Any]        # 冲突检测结果
audit_result: Dict[str, Any]           # 反方审计结果
```

### 工作流变更

```
原流程：
Supervisor → Match/Career → SynthesisGuard → 输出

新流程：
Supervisor → Match → RedTeamAuditor → SynthesisGuard → 输出
         ↘ ConflictDetector → SynthesisGuard
```

### match_agent 双路融合

```
Neo4j图谱查询（优先）
    ↓ 失败
SQLite查询（降级）
    ↓
结果融合去重
```

## 🚀 快速开始

### 1. 启动Neo4j

```bash
docker run -d --name neo4j -p 7474:7474 -p 7687:7687 -e NEO4J_AUTH=neo4j/password neo4j:5.15-community
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 导入数据

```bash
python scripts/import_neo4j.py
```

### 4. 启动服务

```bash
python -m api.main
```

### 5. 访问问卷

```
http://127.0.0.1:5000 → 点击"填写问卷"
```

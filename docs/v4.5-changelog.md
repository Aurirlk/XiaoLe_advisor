## 新增功能

### 1. 用户认证系统

**新增文件:**
- `core/auth.py` — JWT Token生成/验证 + bcrypt密码哈希
- `api/routers/auth_router.py` — 认证API端点
- `frontend/components/LoginPage.js` — 登录/注册页面
- `frontend/utils/apiClient.js` — 带Token注入的HTTP客户端

**修改文件:**
- `data/sql_schema/04_user_profiles.sql` — 新增 username/password_hash/role 字段
- `scripts/init_sqlite.py` — SQLite迁移新字段
- `api/main.py` — 注册auth_router
- `frontend/index.html` — 集成LoginPage组件
- `requirements.txt` — 添加 PyJWT/passlib/bcrypt

**API端点:**
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/register` | 注册（手机号+密码+角色） |
| POST | `/auth/login` | 登录（返回JWT Token） |
| GET | `/auth/me` | 获取当前用户信息 |

**角色说明:**
- `student` — 学生（默认）
- `parent` — 家长
- `admin` — 管理员

### 3. RRF混合搜索

**修改文件:**
- `tools/rag_tools.py` — 串行降级改为并行RRF融合

**算法说明:**
```
三路并行检索:
  L0: ChromaDB 向量语义检索
  L1: SQLite FTS5 全文检索
  L2: 关键词混合召回

RRF排序公式:
  score = 1/(60+rank_vector) + 1/(60+rank_fts5) + 1/(60+rank_keyword)
```

**新增测试:**
- `tests/test_hybrid_search.py` — RRF算法单元测试

python -m pytest tests/ -v

### 测试认证系统

```bash
# 1. 注册新用户
curl -X POST http://127.0.0.1:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"phone_number": "13800138000", "password": "test123", "role": "student"}'

# 2. 登录获取Token
curl -X POST http://127.0.0.1:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone_number": "13800138000", "password": "test123"}'

# 3. 使用Token访问受保护接口
curl http://127.0.0.1:8000/auth/me \
  -H "Authorization: Bearer <your_token_here>"
```

## 升级指南

1. 更新代码到V4.5
2. 安装新依赖: `pip install -r requirements.txt`
3. 设置新环境变量（JWT_SECRET, GUGU_API_KEY）
4. 重启服务，数据库会自动迁移新字段
5. 首次登录需注册账号（无预设管理员账号）
